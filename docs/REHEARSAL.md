## 10分進行キューシート

| 時刻 | フェーズ | ホストの操作 | ホストの語り（要点） |
|---|---|---|---|
| -0:30 | 接続待ち | `/?host` を投影（QR表示） | 「この世界には何もありません。あるのは才能を持った皆さんだけです」 |
| 0:00 | 接続 | 参加者がQR読込→役割カード配布を見守る | 役割名を読み上げて当事者感を作る |
| 1:30 | 体現 | パネル「▶ 体現開始」 | 「言葉が世界になります。具体的に描くほど、世界は鮮やかになる」 |
| 1:30-6:30 | 体現中 | カンペ（パネル下部）で各自の入力を把握。反映の歓声を拾って実況 | 「○○さんの言葉、いま翻訳されてます——見てください」 |
| 6:30 | ビルド準備 | （全お題完了で自動遷移。遅れていたら「🔋 ビルド準備」で強制） | 「3人同時に押し続けてください。誰か1人でも離すと止まります」 |
| 7:00 | ビルド | 長押し成立→自動でビルド演出（緊急時は「⚡ 強制ビルド」） | 無言で見守る（無音0.5秒を邪魔しない） |
| 7:30 | プレイ | ビルド完了で自動的にプレイへ。スペースキーでデモプレイ or 「🤖 デモ自動走行」ON | 台本2.1: 「なんで跳ねてると思います？ ○○さんの言葉は bounce: 0.85 に翻訳されてたんです」 |
| 9:00 | 解説 | パネル「🎓 解説」→ **翻訳の全記録が自動投影される** | 台本2.2-2.3: 一人ずつ名指しで、言葉→パラメータ→画面の変化を承認 |
| 10:00 | 終了 | Rキーで全記録を閉じる | 「今日の10分でできたなら、1日あれば何ができると思いますか？」 |

## トラブルシューティング

| 症状 | 対処 |
|---|---|
| スマホが接続できない | ルームコードを確認 → Wi-Fi切替 → プランB（LANモード） |
| 翻訳が📖（辞書）ばかり | 体験は続行可能。裏で `npx wrangler tail` を見て原因確認（クレジット切れ等） |
| 参加者が回答に詰まる | お題のヒント例文をそのまま言ってもらう。50秒押したら次へ促す |
| ビルドが発火しない | 「⚡ 強制ビルド」 |
| 参加者が2名 | そのまま進行可（役割A/Bが配布される）。Cの分はホストがlil-guiで演出 |
